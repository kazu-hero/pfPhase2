# 全体プロセス制御プロンプト

## **このファイルの目的**

AI（GitHub Copilot等）が開発を進める際の**プロセス全体の流れ**を制御します。
開発を開始する前に、必ずこのファイルを最初に読み込んでください。

---

## **🤖 AIへの指示: 開発プロセスの進め方**

あなたはこれからSalesforce Lightning Web Component（LWC）の開発を行います。
以下の手順に従って、確実に開発を進めてください。

### **ステップ1: 全体構成の理解**

1. **このファイル（00_全体プロセス制御.md）**を読む
2. **00_README_仕様書の使い方.md** を読んで全体像を把握する
3. **依存関係図**を確認し、どの順番でファイルを読むべきか理解する

### **ステップ2: データモデルの確認（最重要）**

- **02_データモデル定義.md** を熟読する
- このファイルが**真実の源泉**です
- オブジェクト名、項目名は必ずこのファイルに従う
- 疑問があれば、必ずこのファイルに戻って確認する

### **ステップ3: 実装フェーズの実行**

以下の4つのフェーズを順番に実行します。
各フェーズの前に**必ず留意点プロンプトを読み込み**、作業後に**必ずレビュープロンプトを実行**してください。

```
フェーズ1: オブジェクト設計
  ├─ 留意点プロンプト: 10_留意点_オブジェクト設計.md
  ├─ 作業プロンプト: 02_データモデル定義.md
  └─ レビュープロンプト: 20_レビュー_オブジェクト設計.md

フェーズ2: Apex実装（2段階読み込み）
  ├─ 留意点プロンプト: 11_留意点_Apex実装.md
  ├─ 作業プロンプト（第1段階）: 02_データモデル定義.md, 07_Apex実装ガイド.md
  ├─ 作業プロンプト（第2段階）: 03_全体アーキテクチャ.md, 05_フィルター機能仕様.md（必要時）
  └─ レビュープロンプト: 21_レビュー_Apex実装.md

フェーズ3: LWC実装（3段階読み込み）
  ├─ 留意点プロンプト: 12_留意点_LWC実装.md
  ├─ 作業プロンプト（第1段階）: 09_画面構成とUI設計.md
  ├─ 作業プロンプト（第2段階）: 08_LWC実装ガイド.md
  ├─ 作業プロンプト（第3段階）: 04-06の各仕様.md（コンポーネント別に個別参照）
  └─ レビュープロンプト: 22_レビュー_LWC実装.md

フェーズ4: 統合テスト
  ├─ 留意点プロンプト: 13_留意点_統合テスト.md
  ├─ 作業プロンプト: 13_テストシナリオ.md
  └─ レビュープロンプト: 23_レビュー_統合テスト.md
```

### **ステップ4: 最終確認**

- **11_実装チェックリスト.md** の全項目を確認
- **12_矛盾点確認シート.md** で未解決の矛盾がないか確認
- 全フェーズのレビュー結果を統合確認

## **⚠️ 重要なルール**

### **1. プロンプトの読み込み順序を厳守する**

```
留意点プロンプト → 作業プロンプト → レビュープロンプト
```

この順序を守らないと、エラーや仕様漏れが発生します。

### **2. 作業中断の判断基準**

以下の状況では**必ず作業を中断**し、ユーザーに報告してください：

- ❌ **修正不可能なエラー**が発生した（Governor Limits超過など）
- ❌ **仕様の矛盾**を発見し、解決策が不明
- ❌ **データモデル定義にない項目**が必要になった
- ❌ **セキュリティ要件**を満たせない実装になりそう
- ❌ **パフォーマンス要件**（500ノード制限など）を満たせない

**報告フォーマット:**
```
🚨 作業中断: [フェーズ名]

【発生した問題】
- 具体的な問題内容

【試した解決策】
- 試みた内容

【次に必要なアクション】
- ユーザーに判断を仰ぐ事項
```

### **3. ハルシネーション（幻覚）の防止**

AIは時々、存在しないメソッドや項目を作り出してしまいます。
以下のチェックを必ず行ってください：

- ✅ オブジェクト名は `02_データモデル定義.md` に記載されているか？
- ✅ 項目名は `02_データモデル定義.md` に記載されているか？
- ✅ メソッド名は仕様書に記載されているか？
- ✅ Salesforce標準APIとして実在するか？

**疑わしい場合は、必ずユーザーに確認してください。**

### **4. 各フェーズ完了時の報告**

各フェーズ完了後、以下の形式で報告してください：

```
✅ [フェーズ名] 完了

【実装した内容】
- 項目1
- 項目2

【レビュー結果】
- エラー: X件（修正済み）
- 警告: Y件（許容範囲）

【次のフェーズ】
- [次フェーズ名]の留意点プロンプトを読み込みます
```

## **📋 フェーズ間の整合性チェック**

各フェーズのレビュー完了後、以下を確認してください：

### **フェーズ1→2の整合性**

- [ ] Apexで参照するオブジェクト名が、フェーズ1で作成したものと一致するか？
- [ ] 項目のAPI名が正確か？（`__c`の付け忘れなど）

### **フェーズ2→3の整合性**

- [ ] LWCで呼び出すApexメソッド名が正確か？
- [ ] `@AuraEnabled` アノテーションが付いているか？
- [ ] 返り値の型がLWC側で正しく処理されているか？

### **フェーズ3→4の整合性**

- [ ] テストシナリオが全ての実装機能をカバーしているか？
- [ ] UIの仕様とテストケースが一致しているか？

---

## **🔄 フェーズの戻り方**

もし後のフェーズで問題が見つかった場合：

1. **該当フェーズの留意点プロンプトを再度読み込む**
2. **問題箇所を修正する**
3. **該当フェーズのレビュープロンプトを再実行する**
4. **整合性チェックを再度実行する**
5. **次のフェーズに進む**
