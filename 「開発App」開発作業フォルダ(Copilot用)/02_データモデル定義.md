## **1. 主要オブジェクト（5種類）**

### **1.1 Dev_Service__c（ユーザサービス）**

**用途**: ユーザーからの問い合わせやサポート要求を管理（ケースオブジェクト的な役割）

**API Name**: `Dev_Service__c`

**主要項目**:
| 項目名 | API Name | データ型 | 必須 | 説明 |
|--------|----------|----------|------|------|
| サービス名 | Name | テキスト(80) | ✓ | サービスの名称（自動採番可） |
| ステータス | Status__c | 選択リスト | ✓ | OPEN / CLOSE |
| 担当者 | OwnerId | 参照(User) | ✓ | Salesforce標準項目 |
| ログ番号 | logNo__c | テキスト | | ログ管理用番号 |
| 内容 | body__c | リッチテキスト | | ご要望・ご質問の内容（HTML形式） |
| 課題数 | countIssue__c | 集計(COUNT) | | 関連課題数（自動集計） |
| ステータスシグナル | StatusSignal__c | 数式(Text) | | ステータス画像表示用数式 |

**選択リスト値（Status__c）**:
- `OPEN` - 対応中（デフォルト）
- `CLOSE` - 完了・クローズ

**⚠️ 注意**: Dev_Service__c のみ、問い合わせ管理の性質上、独自のステータス値を使用します。他の4オブジェクト（Issue, TechActionPlan, FunctionRequirements, Dev_Ticket）は TicketStatus グローバル値セット（TODO/WIP/DONE）を使用します。

**リレーション**:
- **下流**: Relation_Support2Issue__c を経由して Issue__c と多対多

## **3. データフロー図**

```
[Dev_Service__c]
        │
        │ Relation_Support2Issue__c
        │ (Dev_Service__c ⇔ Issue__c)
        ↓
    [Issue__c]
        │
        │ Relation_Issue2Plan__c
        │ (Issue__c ⇔ TechActionPlan__c)
        ↓
[TechActionPlan__c]
        │
        │ Relation_Plan2Requirements__c
        │ (TechActionPlan__c ⇔ FunctionRequirements__c)
        ↓
[FunctionRequirements__c]
        │

        │ (FunctionRequirements__c ⇔ Dev_Ticket__c)
        ↓
  [Dev_Ticket__c]
```

## **4. 重要な設計決定事項**

### **主従関係 vs 参照関係の選択**

**推奨：参照関係（Lookup）**

**理由**:
- 多対多の柔軟性を確保
- 削除の連鎖を避ける（例：Issueを削除してもServiceは残る）
- ロールアップ集計は不要

**例外**:
- 特定のビジネスルールで主従関係が必要な場合のみ使用

### **ステータス値の統一**

**TicketStatusグローバル値セット使用オブジェクト**（Issue, TechActionPlan, FunctionRequirements, Dev_Ticket）:
- `TODO` - 未着手
- `WIP` - 作業中（Work In Progress）
- `DONE` - 完了
- `PENDING` - 保留
- `DROP` - 中止

**Dev_Service__c独自ステータス**:
- `OPEN` - 対応中
- `CLOSE` - クローズ

**注意**: 実装では大文字を使用（TODO/WIP/DONE/OPEN/CLOSE）

### **命名規則**

- **オブジェクトAPI名**: CamelCase + `__c`
- **項目API名**: Snake_Case + `__c`
- **中間オブジェクト名**: `Relation_` + 接続先の略称

## **5. SOQLクエリのサンプル**

### **5.1 全ての課題とその関連サービスを取得**
```sql
SELECT Id, Name, Status__c,
    (SELECT Dev_Service__c, Dev_Service__r.Name 
     FROM Relation_Support2Issues__r)
FROM Issue__c
```

### **5.2 特定サービスの下流を全て取得**
```sql
SELECT Id, Issue__c, Issue__r.Name, Issue__r.Status__c
FROM Relation_Support2Issue__c
WHERE Dev_Service__c = :serviceId
```
