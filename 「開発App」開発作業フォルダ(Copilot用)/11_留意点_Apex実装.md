## **⚠️ よくあるミス**

### **ミス1: `with sharing` の付け忘れ**

❌ **NG例:**
```apex
public class RelationGraphController {
    // セキュリティリスク！
}
```

✅ **正しい例:**
```apex
public with sharing class RelationGraphController {
    // 安全
}
```

### **ミス2: N+1問題**

❌ **NG例:**
```apex
for (Dev_Service__c service : services) {
    List<Relation_Support2Issue__c> relations = [
        SELECT Issue__c FROM Relation_Support2Issue__c 
        WHERE Dev_Service__c = :service.Id
    ]; // ループ内でSOQL！
}
```

✅ **正しい例:**
```apex
Set<Id> serviceIds = new Set<Id>();
for (Dev_Service__c service : services) {
    serviceIds.add(service.Id);
}
// 一括で取得
List<Relation_Support2Issue__c> relations = [
    SELECT Issue__c, Dev_Service__c 
    FROM Relation_Support2Issue__c 
    WHERE Dev_Service__c IN :serviceIds
];
```

### **ミス3: オブジェクト名のハルシネーション**

❌ **NG例:**
```apex
// 存在しないオブジェクト名
List<Service__c> services = [SELECT Id FROM Service__c];
```

✅ **正しい例:**
```apex
// 02_データモデル定義.md 通り
List<Dev_Service__c> services = [SELECT Id FROM Dev_Service__c];
```

**必ず `02_データモデル定義.md` を参照してください。**

## **💡 実装のヒント**

### **ヒント1: 再帰的なデータ取得**

多対多のリレーションを辿るには再帰的な処理が必要です：

```apex
private void traverseRelations(Set<Id> recordIds, Set<Id> visited, Integer depth) {
    if (depth > 10 || recordIds.isEmpty()) return; // 深さ制限
    
    visited.addAll(recordIds);
    
    // 次の階層を取得
    Set<Id> nextIds = getNextLevelRecords(recordIds);
    nextIds.removeAll(visited); // 訪問済みを除外
    
    traverseRelations(nextIds, visited, depth + 1);
}
```

### **ヒント2: ノードとエッジのJSON構築**

Cytoscape.js用のJSON形式：

```apex
Map<String, Object> node = new Map<String, Object>{
    'data' => new Map<String, Object>{
        'id' => recordId,
        'label' => recordName,
        'objectType' => 'Dev_Service__c',
        'status' => 'OPEN'  // または 'CLOSE', 'TODO', 'WIP', 'DONE'
    }
};
```
