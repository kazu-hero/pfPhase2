# レビュー: オブジェクト設計の確認

## **⚠️ 重要: このプロジェクトは既に実装済みです**

## **このレビューの目的**

既存のオブジェクトと項目が、仕様書通りに正しく実装されているかを確認します。差異があれば報告します。

---

## **🤖 AIへの指示: レビューの実施方法**

### **レビューの進め方**

1. このファイルの各チェック項目を**順番に**確認する
2. 差異を発見したら、**報告する（勝手に修正しない）**
3. 全ての差異を記録する
4. 全てのチェックが完了するまで確認を続ける

### **レビュー結果の記録**

以下の形式で結果を記録してください：

```
📊 レビュー結果: オブジェクト設計確認

【チェック項目数】 全X項目
【確認完了項目】 X件
【仕様との一致】 Y件
【差異発見】 Z件

【発見した差異】
1. 差異: Dev_Service__cのStatus__c項目の選択リスト値
   仕様: TODO, WIP, DONE
   実装: OPEN, CLOSE
   判断: これは仕様通り（Dev_Service__cは独自ステータス）

2. 差異: [内容]
   仕様: [仕様書の記載]
   実装: [実際の実装]
   判断: [ユーザー確認が必要 / 仕様通り / 実装ミス]
```

- ❌ 誤: `todo`, `wip`, `done`

## **🔍 ハルシネーション（幻覚）のチェック**

AIが存在しない項目やオブジェクトを「確認した」と誤認していないか確認します。

### **チェック方法**

以下の質問に答えてください：

1. **確認したと報告したオブジェクトは、実際にSalesforceに存在しますか？**
   - [ ] はい、全て存在する
   - [ ] いいえ、存在しないものがある → **差異として報告**

2. **確認したと報告した項目は、実際にSalesforceに存在しますか？**
   - [ ] はい、全て存在する
   - [ ] いいえ、存在しないものがある → **差異として報告**

3. **`02_データモデル定義.md` に記載のないオブジェクトや項目が実装されていませんか？**
   - [ ] いいえ、仕様書通りのみ実装されている
   - [ ] はい、追加実装がある → **差異として報告**

## **⚠️ よくある差異パターン**

### **パターン1: ステータス値の大文字・小文字**

**仕様**: TODO, WIP, DONE（大文字）
**実装例**: todo, wip, done（小文字）

**判断**: 実装が小文字の場合は差異として報告

### **パターン2: 項目名の違い**

**仕様**: `content__c`（リッチテキスト）
**実装例**: `Description__c`（ロングテキスト）

**判断**: 項目名が異なる場合は差異として報告し、どちらを正とすべきかユーザーに確認

### **パターン3: データ型の違い**

**仕様**: リッチテキスト/HTML
**実装例**: ロングテキスト

**判断**: データ型が異なる場合は差異として報告
