# 留意点: オブジェクト設計の確認

## **⚠️ 重要: このプロジェクトは既に実装済みです**

このフェーズでは**新規作成ではなく、既存オブジェクトの確認と検証**を行います。

## **このフェーズで行うこと**

既存のSalesforce Custom Objectsとリレーションが仕様通りに実装されているかを確認します。

---

## **🤖 AIへの指示: 確認前に必ず理解すること**

### **1. 既存実装の前提**

以下の9個のオブジェクトは**既に実装済み**です：

**主要オブジェクト（5個）**: `Dev_Service__c`, `Issue__c`, `TechActionPlan__c`, `FunctionRequirements__c`, `Dev_Ticket__c`
**中間オブジェクト（4個）**: `Relation_Support2Issue__c`, `Relation_Issue2Plan__c`, `Relation_Plan2Requirements__c`, `Relation_Requirements2Dev__c`

### **2. データモデル定義の熟読**

- **02_データモデル定義.md** を完全に理解してから確認作業を開始してください
- オブジェクト名、項目名、データ型を**一字一句正確に**記憶してください
- 不明点があれば、作業を開始せず、ユーザーに質問してください

### **3. 確認作業で行うこと**

❌ **絶対にやってはいけないこと**:
- 既存オブジェクトの削除
- 既存項目の削除や変更
- 選択リスト値の変更や削除
- データの削除や変更

✅ **やるべきこと**:
- 既存オブジェクトと `02_データモデル定義.md` の整合性確認
- 不足している項目があれば報告（勝手に追加しない）
- 仕様との差異を発見したら報告（勝手に修正しない）
- 項目のデータ型が正しいか確認

### **4. 確認環境へのアクセス**

- [ ] Salesforce組織にアクセスできますか？
- [ ] Setup → Object Manager を開けますか？
- [ ] オブジェクトと項目を閲覧する権限がありますか？

権限がない場合、ユーザーに報告してください。

## **⚠️ 確認時の注意点**

### **注意1: ステータス値は大文字**

既存実装では以下の**大文字**を使用しています：

**Dev_Service__c**: `OPEN`, `CLOSE`
**他4オブジェクト**: `TODO`, `WIP`, `DONE`

❌ **間違った確認**: 小文字（todo, wip, done）を探す
✅ **正しい確認**: 大文字（TODO, WIP, DONE, OPEN, CLOSE）を確認

### **注意2: 項目名の確認**

**Dev_Service__c**: `body__c` (リッチテキスト)
**他4オブジェクト**: `content__c` (リッチテキスト)

❌ **存在しない項目**: `Description__c`, `Due_Date__c`
✅ **実際の項目**: `body__c`, `content__c`, `countXXX__c` (ロールアップ)

### **注意3: 参照関係の確認**

中間オブジェクトの参照関係は**双方向**です。

例: `Relation_Support2Issue__c`
- `Dev_Service__c` への参照項目 ✅
- `Issue__c` への参照項目 ✅

**両方**が存在することを確認してください。

以下の状況では**必ず報告**してください（勝手に修正しないこと）：

```

【実装の実態】
- 該当オブジェクトが存在しません

【推奨アクション】
ユーザーに確認が必要です。
```

### **状況2: 項目名や型が異なる**

```
⚠️ 差異発見: 項目の不一致

【仕様書の記載】
- オブジェクト: Issue__c
- 項目名: content__c
- データ型: リッチテキスト

【実装の実態】
- 項目名: Description__c
- データ型: ロングテキスト

【推奨アクション】
- 仕様書を修正すべきか？
- 実装を修正すべきか？
ユーザーに確認が必要です。
```

### **状況3: 選択リスト値が異なる**

```
⚠️ 差異発見: ステータス値の不一致

【仕様書の記載】
- ステータス値: TODO, WIP, DONE

【実装の実態】
- ステータス値: todo, wip, done

【推奨アクション】
どちらを正とすべきかユーザーに確認が必要です。
```
