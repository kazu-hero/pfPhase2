# フィルター機能仕様

## **概要**

3種類のフィルター機能により、ユーザーが必要な情報に絞り込んでグラフを表示できるようにします。

---

## **1. ステータスフィルター（必須機能）**

### **目的**
進捗状況に応じてノードを絞り込み、作業中や完了したタスクを可視化する

### **UI設計**

```html
<div class="filter-section">
    <h3>ステータス</h3>
    <lightning-checkbox-group
        label="ステータスで絞り込み"
        options={statusOptions}
        value={selectedStatuses}
        onchange={handleStatusChange}>
    </lightning-checkbox-group>
</div>
```

### **データ構造**

**⚠️ 注意**: Dev_Service__c は独自のステータス値（OPEN/CLOSE）を使用します。

```javascript
// statusOptions（全ステータスを含む）
[
    // 開発チケット系
    { label: '未着手 (TODO)', value: 'TODO' },
    { label: '作業中 (WIP)', value: 'WIP' },
    { label: '完了 (DONE)', value: 'DONE' },
    
    // 問い合わせ系（Dev_Service__c用）
    { label: '対応中 (OPEN)', value: 'OPEN' },
    { label: 'クローズ (CLOSE)', value: 'CLOSE' }
]

// selectedStatuses（デフォルト：全て選択）
['TODO', 'WIP', 'DONE', 'OPEN', 'CLOSE']
```

### **フィルター適用ロジック**

```javascript
applyStatusFilter(nodes, selectedStatuses) {
    return nodes.filter(node => {
        return selectedStatuses.includes(node.data.status);
    });
}
```

### **動作例**

| 選択 | 表示されるノード |
|------|----------------|
| 全て選択 (todo, wip, done, OPEN, CLOSE) | 全オブジェクトを表示 |
| todo, wip, OPEN | 未完了タスク + 対応中の問い合わせ |
| done, CLOSE | 完了タスク + クローズ済み問い合わせ |
| OPEN のみ | Dev_Service__c で対応中のもののみ |
| (なし) | エラー：最低1つ選択必須 |

## **2. オブジェクトタイプフィルター**

### **目的**
特定の階層のみを表示し、関心のある範囲に集中する

### **UI設計**

```html
<div class="filter-section">
    <h3>オブジェクトタイプ</h3>
    <lightning-checkbox-group
        label="表示する階層"
        options={objectTypeOptions}
        value={selectedObjectTypes}
        onchange={handleObjectTypeChange}>
    </lightning-checkbox-group>
</div>
```

### **データ構造**

```javascript
// objectTypeOptions
[
    { label: 'ユーザサービス', value: 'Dev_Service__c' },
    { label: '課題・要望', value: 'Issue__c' },
    { label: '対応施策', value: 'TechActionPlan__c' },
    { label: '機能要件', value: 'FunctionRequirements__c' },
    { label: '開発チケット', value: 'Dev_Ticket__c' }
]

// selectedObjectTypes（デフォルト）
['Dev_Service__c', 'Issue__c', 'TechActionPlan__c', 
 'FunctionRequirements__c', 'Dev_Ticket__c']
```

### **フィルター適用ロジック**

```javascript
applyObjectTypeFilter(nodes, edges, selectedTypes) {
    // ノードをフィルタリング
    const filteredNodes = nodes.filter(node => {
        return selectedTypes.includes(node.data.type);
    });
    
    // エッジをフィルタリング（両端のノードが存在する場合のみ）
    const nodeIds = new Set(filteredNodes.map(n => n.data.id));
    const filteredEdges = edges.filter(edge => {
        return nodeIds.has(edge.data.source) && 
               nodeIds.has(edge.data.target);
    });
    
    return { nodes: filteredNodes, edges: filteredEdges };
}
```

### **注意事項**

- **最低1つは選択必須** - 全て未選択はエラー
- 非表示階層をスキップして、表示中のノード間を直接エッジで接続（将来実装検討）

## **3. 起点レコードフィルター（重要機能）**

### **目的**
特定のレコードを起点に、そこから派生する上流・下流の関連レコードのみを表示する

### **UI設計**

```html

    <lightning-radio-group

        options={originObjectOptions}
        value={selectedOriginType}
        onchange={handleOriginTypeChange}>
    </lightning-radio-group>
    
    <!-- レコード検索 -->
    <lightning-input
        label="レコード検索"
        placeholder="レコードを検索..."
        value={searchTerm}
        onchange={handleSearchChange}>
    </lightning-input>
    
    <!-- 検索結果リスト -->
    <lightning-listbox
        options={searchResults}
        value={selectedRecordId}
        onchange={handleRecordSelect}>
    </lightning-listbox>
    
    <!-- 表示範囲オプション -->
    <lightning-checkbox-group
        label="表示範囲"
        options={rangeOptions}
        value={selectedRange}
        onchange={handleRangeChange}>
    </lightning-checkbox-group>
    
    <!-- 適用/クリアボタン -->
    <lightning-button label="適用" onclick={applyOriginFilter}></lightning-button>
    <lightning-button label="クリア" onclick={clearOriginFilter}></lightning-button>
```

### **データ構造**

## **4. フィルターの組み合わせ**

### **適用順序**

```javascript
applyAllFilters(rawData, filters) {
    let result = rawData;
    
    // 1. 起点レコードフィルター（指定されている場合）
    if (filters.originRecord) {
        result = this.applyOriginFilter(result, filters.originRecord);
    }
    
    // 2. オブジェクトタイプフィルター
    if (filters.objectTypes) {
        result = this.applyObjectTypeFilter(result, filters.objectTypes);
    }
    
    // 3. ステータスフィルター
    if (filters.statuses) {
        result = this.applyStatusFilter(result, filters.statuses);
    }
    
    // 4. 孤立ノードの除外
    result = this.removeIsolatedNodes(result);
    
    return result;
}
```

### **孤立ノード除外ロジック**

```javascript
removeIsolatedNodes(graphData) {
    const connectedNodeIds = new Set();
    
    // エッジで接続されているノードIDを収集
    graphData.edges.forEach(edge => {
        connectedNodeIds.add(edge.data.source);
        connectedNodeIds.add(edge.data.target);
    });
    
    // 接続されているノードのみを残す
    const filteredNodes = graphData.nodes.filter(node => {
        return connectedNodeIds.has(node.data.id) || 
               graphData.edges.length === 0; // エッジが0の場合は全ノード表示
    });
    
    return { nodes: filteredNodes, edges: graphData.edges };
}
```

## **5. フィルタークリア機能**

### **実装**

```javascript
clearAllFilters() {
    // デフォルト状態に戻す
    this.selectedStatuses = ['todo', 'wip', 'done'];
    this.selectedObjectTypes = [
        'Dev_Service__c', 
        'Issue__c', 
        'TechActionPlan__c',
        'FunctionRequirements__c', 
        'Dev_Ticket__c'
    ];
    this.selectedOriginType = null;
    this.selectedRecordId = null;
    this.selectedRange = ['upstream', 'downstream'];
    
    // グラフ再描画
    this.loadGraphData();
}
```

## **6. Apex側の実装**

### **メソッドシグネチャ**

```apex
@AuraEnabled(cacheable=true)
public static GraphData getFilteredGraph(FilterCriteria criteria) {
    // criteria:
    //   - objectType (String): 起点のオブジェクトタイプ
    //   - recordId (Id): 起点のレコードID
    //   - includeUpstream (Boolean): 上流を含むか
    //   - includeDownstream (Boolean): 下流を含むか
    //   - statuses (List<String>): ステータスフィルター
    //   - objectTypes (List<String>): オブジェクトタイプフィルター
}
```

### **フィルター適用**

```apex
// Apexでステータスフィルターを適用
String statusCondition = '';
if (criteria.statuses != null && !criteria.statuses.isEmpty()) {
    statusCondition = 'Status__c IN :statuses';
}

// クエリ例
List<Issue__c> issues = Database.query(
    'SELECT Id, Name, Status__c FROM Issue__c ' + 
    (String.isNotBlank(statusCondition) ? 'WHERE ' + statusCondition : '')
);
```

## **7. エラーハンドリング**

### **バリデーション**

```javascript
validateFilters() {
    // ステータス: 最低1つ選択
    if (this.selectedStatuses.length === 0) {
        this.showToast('エラー', 'ステータスを最低1つ選択してください', 'error');
        return false;
    }
    
    // オブジェクトタイプ: 最低1つ選択
    if (this.selectedObjectTypes.length === 0) {
        this.showToast('エラー', 'オブジェクトタイプを最低1つ選択してください', 'error');
        return false;
    }
    
    // 起点レコード: オブジェクトタイプとレコードIDの両方が必要
    if (this.selectedOriginType && !this.selectedRecordId) {
        this.showToast('エラー', 'レコードを選択してください', 'error');
        return false;
    }
    
    return true;
}
```
