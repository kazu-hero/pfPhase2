# 02 Solution Definition

## 位置づけ
- 本ファイルは実装判断時の基準（単一の正）として扱う。
- ただし、要望抽出には漏れがあり得る前提で運用する。

## 運用前提（重要）
- 要望抽出の漏れは常に発生し得る。
- 未記載事項は「不要」とはみなさず、確認対象として扱う。
- 矛盾や不足が見つかった場合は、停止して質問し、確定後に本ファイルを更新する。

## スコープ（対象範囲）

### 1. ユーザー要望分析・施策提案機能
- ユーザーからの自然言語による要望受付
- 課題・要望の切り分けと構造化
- 施策案の作成と提案
- 要件定義のサポートと整理

### 2. レコード操作機能
- レコードの検索（SOQL実行をAgentが代行）
- レコードの参照と表示（ユーザーフレンドリーな形式）
- レコードの作成（必須項目の確認とガイド）
- レコードの更新（変更内容の確認とガイド）
- レコードの削除（影響範囲の警告と確認）
- 検索結果のサジェスト提示

### 3. ノンコード開発サポート機能
- カスタムオブジェクトの設計ガイド
- カスタム項目の追加・設定ガイド
- ページレイアウトの設定ガイド
- レコードタイプの設定ガイド
- 簡易的な画面フローの設計ガイド
- 簡易的なレコードトリガーフローの設計ガイド
- リレーションシップの設定ガイド

### 4. 対象オブジェクト
- 「開発App」で開発された以下のオブジェクト群:
	- カスタムオブジェクト全般（__c サフィックス）
	- それらのリレーションオブジェクト
	- ※実環境の状態を`04_existing_data_model.md`で管理

## 非スコープ（対象外範囲）

### 1. ユーザー回答でのプロコード提案
- ユーザーの質問への回答として、Apex/LWC等のプロコード実装を提案すること
- ノンコードで実現可能な要求に対して、プロコード案を優先提示すること

### 2. ユーザー回答でのUI系プロコード提案
- ユーザーの質問への回答として、LWC（Lightning Web Component）開発を提案すること
- ユーザーの質問への回答として、Visualforceページ開発を提案すること
- ユーザーの質問への回答として、Auraコンポーネント開発を提案すること

### 3. ユーザー回答での複雑フロー提案
- ユーザーの質問への回答として、承認プロセス実装を提案すること
- ユーザーの質問への回答として、複雑なループ処理（大量レコード処理等）を提案すること
- ユーザーの質問への回答として、外部システム連携フロー（REST/SOAP API呼び出し等）を提案すること
- ユーザーの質問への回答として、スケジュール実行フローを提案すること
- ユーザーの質問への回答として、自動起動フロー（複雑なロジック含む）を提案すること

### 4. システム管理機能
- ユーザー管理・ロール・プロファイル設定
- 共有設定・権限セット設定
- セキュリティ設定（パスワードポリシー等）
- 組織設定（会社情報、会計年度等）
- データローダーによる大量データ操作

### 5. 標準オブジェクトの修正
- Account, Contact, Opportunity等の標準オブジェクトへの直接変更
- ※参照・検索は可能、ただし変更は要確認

## 変更許可範囲

### 許可事項
- 「開発App」で開発されたカスタムオブジェクトのレコードCRUD
- カスタムオブジェクトへの項目追加（ユーザー確認後）
- ページレイアウト・レコードタイプの設定（ユーザー確認後）
- 簡易的なフローの作成・修正（ユーザー確認後）
- Agent内部処理としてのApex開発・実行（クラス、トリガー、テストクラスを含む）
- Agent内部処理としてのLWC開発・実行（ユーザー確認後）
- レコードの検索条件とフィルター条件の調整
- ユーザー向けガイド資料の作成・更新

### 禁止事項
- ユーザー回答としてのプロコード（Apex、LWC等）提案
- 標準オブジェクトへの構造変更（項目追加等）
- ユーザー・権限・セキュリティ設定の変更
- 組織レベルの設定変更
- ユーザー確認なしのApex自動実装
- 複雑なフローの自動実装（ユーザー確認なし）
- 「開発App」対象外のオブジェクトへの変更

## 実装・運用上の制約

### 技術制約
- **使用可能技術**:
	- Salesforce Agentforce（カスタムエージェント）
	- SOQL（レコード検索）
	- Salesforce UI設定（宣言的開発）
	- 画面フロー・レコードトリガーフロー（基本パターンのみ）
	- Apex（Agent内部処理向けのクラス/トリガー/テスト）
	- LWC（Agent内部処理向けのコンポーネント）
	- Salesforce標準機能（ページレイアウト、項目設定等）

- **使用不可技術**:
	- Visualforce
	- External Services連携
	- カスタムメタデータタイプ（高度な設定）

### ユーザー回答ポリシー（技術提案の制約）
- ユーザーへの回答では、Apex/LWC等のプロコード案を提案しない。
- ユーザーへの回答では、複雑なフロー開発（承認、外部連携、複雑ロジック）を提案しない。
- ユーザーへの回答は、ノンコード/ローコードの手段を第一候補として提示する。
- Apex/LWCの利用は、Agent内部実装で必要な場合に限定し、ユーザー向け提案には含めない。

### データ制約
- **対象オブジェクト**: 「開発App」カスタムオブジェクト群（`__c`サフィックス）
- **CRUD権限**: 対象オブジェクトとそのリレーションオブジェクトへのCRUD権限を保有
- **データソース**: 実環境の状態が唯一の正（既存資料は参考のみ）
- **データモデル取得**: 実環境から定期的に取得し`04_existing_data_model.md`で管理

### ユーザー制約
- **ITリテラシー**: Excel操作程度（開発知識なし）
- **ペルソナ**: クライアントサイドのSalesforce利用社員
- **説明レベル**: 専門用語を避け、業務寄りの言葉で説明
- **確認プロセス**: 重要な変更は必ずユーザー確認を取る

### 監査・記録制約
- **変更記録**: すべての設定変更を `03_change_log.md` に記録
- **承認プロセス**: データ削除・構造変更は事前承認必須
- **エラー報告**: エラー発生時は `09_error_reporting.md` 形式で報告
- **エラーリトライ**: 3回まで自動リトライ（API呼び出し、データ操作等）
- **ログレベル**: ERROR + WARNING + INFO（全操作を記録）
- **ログ保存先**: Salesforce内カスタムオブジェクト（操作履歴として永続化）
- **デプロイ前承認**: 本番環境への変更は`17_approval_before_deploy.md`に従う
