# 03 Change Log

## 方針
- 本ファイルは「ファイル差分の記録」ではなく、「ユーザーとのコミュニケーションと決定方針」の記録として扱う。
- 実際の資料変更・実装変更の差分はGit（Commit/PR）を正とする。
- コンテキスト開放後でも判断経緯を再現できる情報を残す。

## 最小記録項目（必須）
- 日時
- フェーズ/作業コンテキスト
- ユーザーからの指示・要望（要約）
- AIからの提案内容（要約）
- 決定した方針（採用/不採用を含む）
- 決定理由
- 未解決事項/保留事項
- 次アクション
- 承認/確認の有無

## Git/GitHub 連携項目（任意）
- Branch
- PR番号/URL
- Commit SHA
- マージ方式（squash/merge/rebase）

## 記録例（任意）
```markdown
### 2026-03-08 [作業タイトル]
- フェーズ: [要件整理/設計/実装/テスト/リリース準備]
- ユーザー指示: [何を求められたか]
- AI提案: [提案した方針・選択肢]
- 決定方針: [最終的に採用した方針]
- 決定理由: [なぜその方針にしたか]
- 保留事項: [未解決の論点]
- 次アクション: [次に実施すること]
- 承認: [OK/差し戻し/保留]
- Branch: [feature/...]
- PR: [#123 または URL]
- Commit: [SHA]
- 備考: [必要なら]
```

## 注意
- 変更内容の詳細差分をここに重複記録しない（Gitに一元化）。
- 形式よりも、判断経緯の追跡可能性と再現可能性を優先する。
