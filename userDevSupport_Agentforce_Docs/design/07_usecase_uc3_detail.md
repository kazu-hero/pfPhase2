## 1. ユースケース概要

### 1.1 概要
ユーザーが「ノンコード開発の方法」を質問したときに、Agentが段階的にガイドを提供：

- **オブジェクト設計**: カスタムオブジェクト・項目の作成方法
- **ページレイアウト**: 画面設計・項目配置
- **リレーションシップ**: オブジェクト間の関連付け
- **フロー**: 基本的なプロセス自動化（承認フロー、画面フロー）
- **検証ルール**: データ品質維持

### 1.2 アクター
- **Primary**: 社内ユーザー（ノンコード開発初心者）
- **Secondary**: Agentforce Agent

### 1.3 提供ガイド範囲
✅ 提供範囲:
- カスタムオブジェクト作成
- カスタム項目作成（全項目タイプ）
- ページレイアウト設定
- レコードタイプ設定
- ルックアップ・主従関係設定
- 基本的なフロー（承認・画面フロー）
- 検証ルール・入力規則
- レポート作成（基本）

❌ 範囲外（スコープ外通知 P-005パターン適用）:
- Apex開発
- LWC開発
- 複雑なフロー（ループ処理、複数オブジェクト横断）
- Experience Cloudカスタマイズ
- Visualforceページ

### 4.2 ガイド提供例: カスタムオブジェクト作成

#### ステップ1: 概要
```
【ガイド】カスタムオブジェクトの新規作成

このガイドでは、新しいカスタムオブジェクトを作成する手順を説明します。

【所要時間】約5分

【事前確認】
✅ システム管理者権限が必要です
✅ オブジェクト名は英数字で命名（後で変更できません）

【全5ステップ】
1. セットアップ画面を開く
2. オブジェクトマネージャーに移動
3. 新規カスタムオブジェクトを作成
4. 基本情報を入力
5. 保存して完了

準備ができたら「次へ」と入力してください。
```

#### ステップ2: 詳細手順
```
【ステップ 1/5】セットアップ画面を開く

1. 画面右上の⚙️（ギア）アイコンをクリック
2. 「セットアップ」を選択

✅ セットアップ画面が表示されます。

完了したら「次へ」と入力してください。
（前の手順に戻る場合は「戻る」と入力）
```

#### ステップ3: オブジェクトマネージャー
```
【ステップ 2/5】オブジェクトマネージャーに移動

1. 左側のメニューから「オブジェクトマネージャー」をクリック

✅ 既存のオブジェクト一覧が表示されます。

完了したら「次へ」と入力してください。
```

#### ステップ4: 新規作成
```
【ステップ 3/5】新規カスタムオブジェクトを作成

1. 画面右上の「作成」ドロップダウンをクリック
2. 「カスタムオブジェクト」を選択

✅ カスタムオブジェクト作成画面が開きます。

完了したら「次へ」と入力してください。
```

#### ステップ5: 基本情報入力
```
【ステップ 4/5】基本情報を入力

以下の項目を入力してください：

【必須項目】
- **表示ラベル**: オブジェクトの表示名（日本語OK）
  例: 「商談管理」「顧客情報」
- **表示ラベル（複数形）**: 複数形の表示名
  例: 「商談管理」「顧客情報」
- **オブジェクト名**: API参照名（英数字のみ）
  例: Deal_Management, Customer_Info

【推奨設定】
- ✅ タブスタイルを選択（アイコンを選ぶ）
- ✅ 「レポート可」にチェック
- ✅ 「活動」にチェック（タスク・メモを関連付け可能に）

入力したら「次へ」と入力してください。
```

#### ステップ6: 完了
```
【ステップ 5/5】保存して完了

1. 画面下部の「保存」ボタンをクリック

✅ カスタムオブジェクトが作成されました！

【次のステップ】
新しく作成したオブジェクトに項目を追加しましょう。

1. カスタム項目の追加ガイドを見る
2. ページレイアウトを設定する
3. 別のオブジェクトを作成する

次に何をしますか？番号で選択してください。
```

### 5.1 フロー図


### 5.2 ユーザー入力例
```
「ルックアップ関係とは？」
```

### 5.3 Agent応答例
```
【用語説明】ルックアップ関係

ルックアップ関係は、2つのオブジェクトを緩やかに関連付ける方法です。

【特徴】
- 親レコードを削除しても子レコードは残る
- 子レコードから親レコードを参照できる
- 必須・任意を選択可能

【具体例】
「商談」オブジェクトと「顧客」オブジェクトの関係：
- 商談から顧客を参照できる
- 顧客を削除しても商談は残る

【主従関係との違い】
| 比較項目 | ルックアップ | 主従関係 |
|---------|------------|---------|
| 親削除時 | 子は残る | 子も削除される |
| 必須/任意 | 選択可能 | 必須 |
| 積み上げ集計 | 不可 | 可能 |

【関連ガイド】
1. ルックアップ関係の作成方法を見る
2. 主従関係との使い分けガイドを見る

次に何をしますか？
```

## 7. エラーハンドリング

### 7.1 トピック特定失敗
| エラーシナリオ | エラータイプ | 処理 |
|------------|------------|------|
| トピック不明 | E-001 | P-003適用、候補トピック提示 |
| スコープ外要求 | E-005 | P-005適用、範囲外通知と代替案 |
| 曖昧な質問 | E-001 | P-003適用、質問の言い換え促進 |

### 7.2 ガイド提供時のエラー
| エラーシナリオ | エラータイプ | 処理 |
|------------|------------|------|
| ガイドが存在しない | E-003 | P-003適用、類似ガイド提示 |
| ナレッジベース接続失敗 | E-007 | リトライ実行、失敗時はE-009通知 |

### 7.3 スコープ外要求の例
```
ユーザー: 「Apexでトリガーを作りたい」

Agent:
【スコープ外のご要望】

Apex開発のご要望ですね。

申し訳ございませんが、このAgentでは**Apex開発のガイドは提供できません**。
Apexはコーディングが必要な高度な開発手法のため、このAgentのサポート範囲外です。

【代替案】
以下の方法で同様の機能を実現できる可能性があります：

1. **フロー（Flow）の利用**
   - ノンコードでレコード作成時の自動処理が可能です
   - 基本的な条件分岐や項目更新に対応

2. **検証ルール**
   - データ入力時のチェックルールを設定できます

3. **ワークフロールール**
   - シンプルな自動化処理に適しています

どれかの代替方法について詳しく知りたいですか？
```
