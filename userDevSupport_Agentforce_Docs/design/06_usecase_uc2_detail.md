## 1. ユースケース概要

### 1.1 概要
ユーザーがAgentを通じて「開発App」オブジェクト群のレコードに対してCRUD操作を実行：
- **Create**: 新規レコード作成
- **Read**: レコード検索・詳細参照
- **Update**: 既存レコード更新
- **Delete**: レコード削除

### 1.2 アクター
- **Primary**: 社内ユーザー
- **Secondary**: Agentforce Agent

### 1.3 対象オブジェクト
- Dev_Service__c
- Issue__c
- TechActionPlan__c
- FunctionRequirements__c
- Dev_Ticket__c
- 4つのJunction Objects（Relation_*）

## 2. サブユースケース一覧

### 2.1 UC-2-1: レコード検索
ユーザーが検索条件を指定してレコードを検索

### 2.2 UC-2-2: レコード詳細参照

### 2.3 UC-2-3: レコード作成

### 2.4 UC-2-4: レコード更新

### 2.5 UC-2-5: レコード削除
不要なレコードを削除

## 3. UC-2-1: レコード検索詳細設計

### 3.1 フロー図
```

    ↓

    ↓
[Agent: 検索結果表示] ← P-002パターン
    ↓

```

### 3.4 SOQL設計
```apex
// パターン1: 全件検索
SELECT Id, Name, Status__c, category__c, OwnerId, CreatedDate
FROM Issue__c
ORDER BY CreatedDate DESC
LIMIT 100

// パターン2: ステータス検索
SELECT Id, Name, Status__c, category__c, Owner.Name, CreatedDate
FROM Issue__c
WHERE Status__c = 'WIP'
ORDER BY CreatedDate DESC
LIMIT 100

// パターン3: キーワード検索
SELECT Id, Name, Status__c, category__c, Owner.Name, CreatedDate
FROM Issue__c
WHERE Name LIKE '%売上%'
ORDER BY CreatedDate DESC
LIMIT 100

// パターン4: 複合条件検索
SELECT Id, Name, Status__c, category__c, Owner.Name, CreatedDate
FROM Issue__c
WHERE Status__c = 'TODO'
    AND Owner.Name LIKE '%山田%'
ORDER BY CreatedDate DESC
LIMIT 100
```

## 4. UC-2-2: レコード詳細参照詳細設計

### 4.1 フロー図
```
[ユーザー: レコード指定]
    ↓
[Agent: レコード詳細取得] ← getRecordDetails Action
    ↓
[Agent: 詳細表示] ← P-002パターン
    ↓
[Agent: 関連レコード取得]（オプション）
    ↓
[Agent: 次アクション提案] ← P-007パターン
```

### 4.2 ユーザー入力例
```
「1番目のレコードの詳細を見せてください」
```
または
```
「レコードID a0B1234567890 の詳細を表示してください」
```

### 4.3 Agent応答例
```
【レコード詳細】売上レポートの改善

オブジェクト: 課題・要望（Issue__c）

【基本情報】
- 課題名: 売上レポートの改善
- ステータス: WIP（作業中）
- カテゴリ: レポート改善
- 担当者: 佐藤花子
- 作成日: 2026年3月1日 10:30
- 最終更新日: 2026年3月7日 15:45
- レコードID: a0B1234567890

【詳細内容】
月次売上レポートに顧客別の内訳を追加する。
現状は全体の売上のみ表示されており、顧客ごとの貢献度が把握できない。

【集計情報】
- 関連サポート数: 2件
- 関連施策数: 1件

【関連レコード】
■ ユーザサービス（Dev_Service__c）
1. レポート改善要望（ID: a0A1234567890）
2. 売上分析強化（ID: a0A1234567891）

■ 対応施策（TechActionPlan__c）
1. カスタムレポート作成（ID: a0C1234567890）

次に何をしますか？
1. このレコードを更新する
2. 関連レコードの詳細を確認する
3. 対応施策を追加する
4. 検索画面に戻る
```

### 4.4 SOQL設計
```apex
// 基本詳細取得
SELECT Id, Name, Status__c, category__c, content__c,
       OwnerId, Owner.Name, CreatedDate, LastModifiedDate,
       countSupport__c, countPlan__c
FROM Issue__c
WHERE Id = :recordId
LIMIT 1

// 関連レコード含む（サブクエリ）
SELECT Id, Name, Status__c, category__c, content__c,
       OwnerId, Owner.Name, CreatedDate, LastModifiedDate,
       (SELECT Id, Name, Dev_Service__c, Dev_Service__r.Name
        FROM Relation_Support2Issue__r
        ORDER BY CreatedDate DESC
        LIMIT 10),
       (SELECT Id, Name, TechActionPlan__c, TechActionPlan__r.Name
        FROM Relation_Issue2Plan__r
        ORDER BY CreatedDate DESC
        LIMIT 10)
FROM Issue__c
WHERE Id = :recordId
LIMIT 1
```

## 5. UC-2-3: レコード作成詳細設計

### 5.1 フロー図
```
[ユーザー: 作成依頼]
    ↓
[Agent: オブジェクト確認]
    ↓
[Agent: 必須項目確認]
    ↓
[ユーザー: 項目値入力]（対話形式）
    ↓
[Agent: 入力内容確認] ← P-001パターン
    ↓（承認）
[Agent: レコード作成] ← createRecord Action
    ↓
[Agent: 完了報告] ← P-006パターン
```

### 5.2 ユーザー入力例
```
「新しい課題を登録したい」
```

### 5.3 Agent応答例（対話形式）

#### ステップ1: オブジェクト確認
```
新しい課題を登録しますね。

【オブジェクト】課題・要望（Issue__c）

必須項目を順番に入力していきましょう。

まず、課題名を教えてください。
（例: データ入力の自動化）
```

#### ステップ2: 項目入力（課題名）
**ユーザー**: 「データ入力の自動化」

```
課題名: データ入力の自動化

次に、ステータスを選択してください：
1. TODO（未着手）- デフォルト
2. WIP（作業中）
3. PENDING（保留）

番号で選択してください。（未指定の場合は「TODO」になります）
```

#### ステップ3: 項目入力（ステータス）
**ユーザー**: 「1」

```
ステータス: TODO（未着手）

次にカテゴリを選択してください（任意）：
1. レポート改善
2. UI改善
3. 業務効率化
4. その他
5. 指定しない

番号で選択してください。
```

#### ステップ4: 項目入力（カテゴリ）
**ユーザー**: 「3」

```
カテゴリ: 業務効率化

最後に、詳細内容を教えてください（任意）。
（例: 現在手動で行っているデータ入力を自動化したい）
```

#### ステップ5: 項目入力（内容）
**ユーザー**: 「現在手動で行っているデータ入力を自動化したい」

```
【確認】レコード作成

以下の内容で「課題・要望（Issue__c）」レコードを作成します：

- 課題名: データ入力の自動化
- ステータス: TODO（未着手）
- カテゴリ: 業務効率化
- 内容: 現在手動で行っているデータ入力を自動化したい
- 担当者: （あなた）

この内容でレコードを作成してよろしいですか？
「はい」「実行」「OK」などで承認いただければ処理を開始します。
```

#### ステップ6: 作成完了
```
【完了】レコード作成

「課題・要望（Issue__c）」レコードを作成しました。

- レコード名: データ入力の自動化
- レコードID: a0B9999999999
- ステータス: TODO（未着手）

次に何をしますか？
1. 関連するユーザサービスを検索する
2. 対応施策を作成する
3. レコードの詳細を確認する
4. 別のレコードを作成する
```

## 8. エラーハンドリング

### 8.1 検索時のエラー
| エラーシナリオ | エラータイプ | 処理 |
|------------|------------|------|
| 無効なオブジェクト名 | E-003 | P-003適用、正しいオブジェクト名提示 |
| 権限不足 | E-004 | P-003適用、権限不足通知 |
| 不正な検索条件 | E-001 | P-003適用、検索条件修正促進 |

### 8.2 作成時のエラー
| エラーシナリオ | エラータイプ | 処理 |
|------------|------------|------|
| 必須項目未入力 | E-001 | P-003適用、再入力促進 |
| 重複レコード | E-002 | P-003適用、代替案提示 |
| 権限不足 | E-004 | P-003適用、権限不足通知 |

### 8.3 更新時のエラー
| エラーシナリオ | エラータイプ | 処理 |
|------------|------------|------|
| レコードが存在しない | E-003 | P-003適用、レコード確認促進 |
| 権限不足 | E-004 | P-003適用、権限不足通知 |
| ロック競合 | E-007 | リトライ実行 |

### 8.4 削除時のエラー
| エラーシナリオ | エラータイプ | 処理 |
|------------|------------|------|
| レコードが存在しない | E-003 | P-003適用、レコード確認促進 |
| 権限不足 | E-004 | P-003適用、権限不足通知 |
| 参照制約違反 | E-006 | P-003適用、削除不可通知 |
