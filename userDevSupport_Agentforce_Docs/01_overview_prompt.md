# 01 Overview Prompt

## 目的
- AIが作業開始前にプロジェクトの全体感を短時間で把握するための概要。

## 最小確認項目

### プロジェクト目的
- Salesforceノンコード開発をサポートするAgentシステムの構築
- ユーザーの要望・課題を分析し、施策作成・要件定義をサポート
- レコードのCRUD操作による業務効率化
- 検索結果の表示とサジェスト機能の提供

### 対象範囲
- ユーザー要望・課題の切り分けと分析
- 施策作成と要件定義のサポート
- SalesforceUI上でのノンコード開発支援（オブジェクト、項目、レイアウト、ページレイアウト等）
- 簡易的なフロー開発支援（画面フロー、レコードトリガーフローの基本パターン）
- Agent内部処理として必要なApex開発・実行
- Agent内部処理として必要なLWC開発・実行
- レコードのCRUD操作（作成、参照、更新、削除）
- レコード検索とサジェスト表示
- 「開発App」成果物オブジェクト群とそのリレーションオブジェクトの操作

### 非対象範囲
- ユーザー回答としてのApex/LWC提案
- ユーザー回答としての複雑なフロー開発提案（承認プロセス、複雑なループ処理、外部連携等）
- Visualforceページ開発
- カスタムアプリケーション開発（プロコード）
- システム管理機能（セキュリティ設定、組織設定等）

### 成果物
- Agentforce用のカスタムエージェント定義
- ユーザー対話フロー設計書
- レコードCRUD操作の実装仕様
- オブジェクト設定・項目定義のサポート資料
- 簡易フロー実装パターン集
- ユーザーガイド・操作手順書

### 主な制約
- **技術制約**: 
	- Salesforce Agentforce機能の範囲内での実装
	- ユーザー向け回答はノンコード・ローコード案のみ提示
	- Apex/LWCはAgent内部処理として利用可（ユーザー回答での提案は禁止）
	- 複雑なフローはユーザー回答での提案を禁止（内部実装時はユーザー確認を必須）
	- 「開発App」オブジェクト群へのCRUD権限に限定
- **ユーザー制約**: 
	- ITリテラシー: Excel操作程度（開発知識なし）
	- クライアントサイドのSalesforce利用社員がペルソナ
- **運用制約**: 
	- 実環境の状態が唯一の正（既存資料は参考）
	- データモデルは実環境から取得

### 依存関係
- 「開発App」開発作業フォルダで開発された成果物オブジェクト群の存在
- Salesforce組織へのアクセス権限
- Agentforce機能の有効化
- レコードCRUD権限の付与

### 完了条件
- ユーザーが自然言語でAgentに要望を伝えられる
- Agentが要望を切り分けて施策提案できる
- Agentがレコードの検索・表示・作成・更新・削除を実行できる
- 簡易的なオブジェクト設定・フロー開発のガイドができる
- 開発知識のないユーザーが独力でノンコード開発を進められる
