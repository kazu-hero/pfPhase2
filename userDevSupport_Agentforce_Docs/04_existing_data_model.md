# 04 Existing Data Model

## 方針
- 開発物に応じて最適な粒度で記載する。
- テンプレートはガイドであり、形式固定はしない。

## 最小記載項目（推奨）
- 対象オブジェクト/テーブル
- 主要フィールド
- 主要リレーション
- 一意制約・識別子
- 更新時の注意点（参照整合性、必須項目など）

## 記載例（任意）
```markdown
## [オブジェクト名]
- 目的: [このデータの役割]
- 主要フィールド: [field_a, field_b, ...]
- リレーション: [親子/参照先]
- 制約: [必須、ユニーク、参照制約]
```

## 参照元
- `force-app/main/default/objects` 配下
- 必要に応じて org schema のエクスポート結果
