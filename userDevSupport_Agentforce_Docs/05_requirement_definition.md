# 05 Requirement Definition

## 方針
- 要件定義の形式は開発物に合わせて最適化する。
- 本テンプレートは最低限の抜け漏れ防止チェックとして使う。

## 最小確認項目（推奨）
- 背景/目的: [なぜ必要か]
- ユーザー価値: [誰にどんな価値があるか]
- 対象範囲: [今回実施する内容]
- 非対象範囲: [今回実施しない内容]
- 受け入れ条件: [完了判定の基準]
- 制約条件: [技術/運用/期限など]
- リスク/未確定事項: [決まっていない事項]

## 確認が必要な項目（初期値）
- [要確認1]
- [要確認2]

## 運用ルール
- 未確定事項がある場合は質問して確定する。
- 確定内容は `02_solution_definition.md` に反映し、必要なら `03_change_log.md` に決定理由とユーザー合意内容を記録する。
