# Agentforce開発 初期設計概要

## 背景

本プロジェクトは、`Dev_Service__c -> Issue__c -> TechActionPlan__c -> FunctionRequirements__c -> Dev_Ticket__c` の5階層データモデルを対象に、オブジェクト横断で内容を要約し進捗を報告するPoCです。初回テストのため、単一Agentで最小機能の成立を優先します。

※ 上記は概念上の階層であり、実データ参照は `Relation_Support2Issue__c` / `Relation_Issue2Plan__c` / `Relation_Plan2Requirements__c` / `Relation_Requirements2Dev__c` のJunction Objectを経由します。

## 目的

- 単一Agentで、指定された `Dev_Service__c` を起点に全体状況を会話で報告できるようにする
- 5階層（Issue, Plan, Requirements, Ticket）の状態を横断集計して提示する
- 初回PoCとして「要約と進捗報告」のみを対象にし、機能を絞って検証する

## 基本方針

- Agentは1つのみ作成する
- 配置は `Dev_Service__c` レコードページとし、開いているレコードのIDがコンテキストに自動注入される
- 集計処理はApex Action `getProgressByDevService(devServiceId)` に集約する
- Agentは要約と進捗報告に専念し、レコード作成や更新は本PoCでは行わない
- 調査、要件、設計成果物は `./Agentforce_DevelopAgent/` 配下に集約する

## 初期構成（単一Agent）

Agent:      `agent a` Progress Reporter
配置場所:   `Dev_Service__c` レコードページ
担当:       `Dev_Service__c` 指定に基づく横断要約・進捗報告

## ログ設計

- ログ詳細は本初期設計の対象外（必要時に別途定義）

## 進捗報告機能

- 開いている `Dev_Service__c` レコードのIDをコンテキストとして自動取得
- Apex Action `getProgressByDevService(devServiceId)` を呼び出し
- 集計は Junction Object を経由して実行する（`Relation_Support2Issue__c` → `Relation_Issue2Plan__c` → `Relation_Plan2Requirements__c` → `Relation_Requirements2Dev__c`）
- 5階層（Issue, Plan, Requirements, Ticket）を横断して `Status__c` 集計結果を会話型で提示
- ステータスは `Dev_Service__c`（`OPEN/CLOSE`）と他4オブジェクト（`TicketStatus`: `TODO/WIP/DONE/PENDING/DROP`）を区別して扱い、PoCでは `NotStarted/InProgress/Done/Blocked` に正規化して表示する
- 応答は「概要」「ステータス件数」「次に確認すべき対象」の3ブロックで返す

## スコープ（概要フェーズ）

対象:
- 単一Agent（`agent a`）の責務と配置
- `getProgressByDevService(devServiceId)` の集計要件
- 会話による横断要約・進捗報告の応答仕様

対象外（本ドキュメント）:
- 複数Agent分割設計
- レコード作成/更新系オートメーション
- ログスキーマの詳細設計
- UI詳細設計、テストケース詳細
