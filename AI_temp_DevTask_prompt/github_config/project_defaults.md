# GitHub Project Defaults

## ブランチ戦略
- main + feature/*

## 承認方針（ソロ開発）
- self-review のみで承認可

## hotfix
- 必要時のみ作成・使用する

## マージ方式
- squash merge

## PRタイトル規約
- prefix 必須
- 許可prefix: `feat:`, `fix:`, `docs:`

## rebase
- 通常運用では使用しない
- 例外時のみ、ユーザー承認で実施

## CI状態
- 未導入
- 導入を検討中

## 禁止事項
- direct push to main
- force push
- hard reset
